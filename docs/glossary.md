# Glossary

## AI Planner

The system component that turns goals or mission context into structured plans. It should not directly control motors.

## Autonomy

The ability of the robot system to make decisions and perform tasks with limited human input.

## Backend

The server-side software that manages missions, robot state, payload state, logs, and communication between the dashboard and ROS 2.

## Dashboard

The user interface used by a human operator to monitor and control the robot system.

## Emergency Stop

A high-priority safety action that immediately stops robot motion or dangerous payload activity.

## Mission

A structured task or set of tasks assigned to the robot system.

## Payload

Any mounted device on the robot that is not part of the base robot platform, such as a camera, chemical detector, vibration sensor, or laser spectrometer.

## Platform Adapter

A software layer that connects the common robot control system to a specific robot vendor API or SDK.

## ROS 2

Robot Operating System 2. A robotics middleware used for communication between robot software components.

## Sensor Fusion

Combining data from multiple sensors to produce a better estimate of the robot state or environment.

## Simulation

A software environment where robot behavior can be tested without using real hardware.
