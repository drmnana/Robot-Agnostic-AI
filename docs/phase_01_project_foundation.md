# Phase 01 - Project Foundation

## Goal

Define the project clearly before building code.

This phase creates the foundation for the entire system: goals, constraints, safety expectations, first use cases, and project organization.

## Why This Comes First

Robotics projects become difficult when hardware, AI, sensors, networking, and user interfaces are all developed at once without a shared structure.

This phase prevents that by deciding what the system is and what version 1 should focus on.

## Main Tasks

- Confirm the first target robot platform.
- Confirm whether version 1 starts in simulation, real hardware, or both.
- Define the first mission scenario.
- Define the first payload or sensor.
- Define autonomy level for version 1.
- Define safety rules from the beginning.
- Create project documentation structure.
- Create the initial GitHub repository.
- Decide naming conventions for packages, services, and documents.

## Recommended Version 1 Scope

Version 1 should be simple and controlled:

- One robot platform
- One simulated robot or one real robot
- One basic mission type
- One or two sensors
- Manual override
- Emergency stop
- Basic mission logging

## Outputs

- Architecture reference document
- Project phase roadmap
- Safety assumptions document
- Initial Git repository
- Initial folder structure
- First mission scenario definition

## Completion Criteria

This phase is complete when we can clearly answer:

- What are we building first?
- Which robot comes first?
- Which sensor comes first?
- What is the first mission?
- What must the robot never be allowed to do?
- Where will the code and documentation live?
